3D Model Viewer for Apple Vision Pro

A short demo created to showcase how an existing VR application on Meta Quest 3 could be adapted to the windows and controller-free interactions on the Apple Vision Pro.

Login experience with on-screen keyboard.

Main dashboard for locating a case (model) to open.

Pop-up confirmation window for loading the case (model).

The user can access a toolbox by raising their wrist, palm towards the user.

The user can bring out any tool via pinch and hold, and place it in the space.

Alternate interaction allows users to tap a tool with their index finger, to spawn in worldspace between the user and model.

3D Digital Twin Visualizer for Ships

Pre-production concepts created for a shipping company, evaluating the condition of their ship compartments and parts that require attention, based on a severity scale. This project dealt with heavy information architecture, requiring synthesis from multiple text-only sources, made into a visually intuitive interface using the client's existing branding.

Concept 1: Default screen with condition score expanded, correlating to severity shown on the 3D model.

Concept 1: An expanded bottom window with additional details and an expanded hierarchy menu to organize reports and parts.

Concept 1: Showing nested information within a section of the bottom window.

Concept 2: A vertically organized concept. This screen shows a pop-up menu when the user right-clicks a part on the model

Concept 2: Right side panel showing additional details of anomalies, with categorized information types and attachments.

Concept 2: Expanded right side panel to fit larger collections of information.

Concept 2: A more compact layout for the condition scores menu.

Concept 2: Right side panel showing anomalies via card format, for quicker access.

AI Conversational Digital Twin

A demo created to explore conversing with AI and receiving answers via 3D model. When the user asks a question, the LLM will show the user the answer using animations on the 3D model, which the user can then complete step by step. Extra care was taken to create naturally flowing conversations with minimal UI obstructions.

The user can tap the button at the bottom of the screen to ask a question, which the LLM logs at the top.

To keep the interface as clean as possible, alerts are given via toasts rather than a static UI window.

A menu is nested away in the left with model options and settings.

Various modals used for switching the model, providing onboarding instructions, and crucial alerts.
